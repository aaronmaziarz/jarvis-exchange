# Chronos Competitive Feedback Analysis

*Compiled: 2026-03-11*
*Purpose: Research findings to inform Chronos product decisions*

---

## Executive Summary

Researched user complaints and feature requests from Motion and Akiflow users. Key insight: **users are frustrated with AI "slop", forced upgrades, and feature bloat**. They want **simple, effective task scheduling** without the fluff.

---

## Motion App Complaints (from Reddit)

### Critical Issues
1. **Auto-scheduling reduced** - Was 90 days, now only 30 days unless you pay for "AI Pro"
2. **Forced expensive upgrades** - Basic features behind expensive paywall
3. **Feature bloat** - Pushing unwanted AI features (AI Docs, AI Meeting Notes)
4. **Terrible customer service**
5. **Steep learning curve** - Took users "a few full days" to learn

### What Users Actually Want
- "I just want my tasks to be scheduled automatically"
- "I don't want AI slop in my face"
- Straightforward solution without unnecessary AI features
- Multiple calendar sync (Google, Outlook)
- Task ordering and dependencies
- Recurring tasks
- Easy setup (not steep learning curve)

---

## Akiflow Complaints (from Reddit)

### Critical Issues
1. **Deleted "free forever" tier** - Users furious about silent removal
2. **Secret price changes** - No transparency
3. **Limited AI** - Mostly just calendar sync, no real auto-scheduling
4. **No native Mac app** - Web-only for desktop
5. **No native iPhone** - Mobile web is okay but not great

### What Users Want
- Better mobile experience (native app)
- More AI automation
- Transparent pricing
- Free tier for casual users

## Feature Requests to Prioritize

Based on user feedback, these features would differentiate Chronos:

### Must-Have (from complaints)
1. **Simple, effective auto-scheduling** - Don't overcomplicate it
2. **Multiple calendar sync** - Google, Outlook, Apple iCloud
3. **Native Mac app** - Not web-only
4. **Native iPhone app** - Not mobile web
5. **Transparent pricing** - No hidden changes
6. **Easy onboarding** - Don't make it take "days" to learn
7. **Free tier** - Even if limited

### Nice-to-Have (differentiators)
1. **Social planning** - Make plans with friends
2. **Flexible event types** - Give users control over what moves
3. **Smart inbox** - AI suggests schedule, user approves
4. **User controls** - Work hours, focus time protection

---

## What NOT to Do (Lessons Learned)

❌ Don't reduce free features to force upgrades  
❌ Don't push AI features users don't want  
❌ Don't remove "forever free" tiers silently  
❌ Don't make the UI take days to learn  
❌ Don't call everything "AI" just for marketing  
❌ Don't ignore customer service  

---

## Chronos Positioning

**Slogan idea:** "Your tasks, scheduled. Simply."

**Key differentiators:**
1. **Simple scheduling** - No AI bloat, just results
2. **Native apps** - Mac + iPhone (not web-only)
3. **Transparent** - No hidden pricing changes
4. **Free tier** - Generous free forever plan
5. **User in control** - AI assists, doesn't override
6. **Social features** - Unique: plan with friends

---

## Implementation Recommendations

### Quick Wins (Easy to Implement)
1. **Multiple calendar sync** - High demand, straightforward
2. **Task duration + deadline** → auto-schedule - Core value
3. **Native iPhone app** - Users want this
4. **Clear onboarding** - First-time user flow

### Differentiators (Competitive Edge)
1. **Social planning** - Neither competitor has this
2. **Flexible event types** - Give users scheduling control
3. **Smart inbox** - But make it optional, not forced
4. **Transparent pricing** - Announce changes publicly

### Avoid
- Don't call everything "AI" if it's just scheduling
- Don't create steep learning curves
- Don't remove free features later
- Don't push features users didn't ask for
